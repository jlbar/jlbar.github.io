CLI Development Environment, Look and Feel

15/Jun 2017

Here, I will detail the installation and configuration of the foundational components comprising the local development environment on my laptop. This post is not meant to be exhaustive, and will focus primarily on look-and-feel items. For simplicity, I will focus on the three primary components of my development environment: the shell (zsh), the terminal multiplexer (tmux), and the editor (vim).

A Note on Color Themes

There exist a dizzying array of available color themes across the applications I use. So many, in fact, that it is easy for me to become inundated by the sheer number of options available. Thus, I will make it simple: use the base16 color theme. Use it everywhere. In order to use the base16 theme in vim, I must make the colors available in my terminal emulator. The base16 theme is pleasant, well-maintained, and available across multiple platforms including vim and Terminal.app.

Terminal.app

I use the OSX Terminal.app terminal emulator. I chose this as opposed to the popular iTerm2 because Terminal.app comes with the OS and is one less dependency to manage. Very little configuration is required of Terminal.app, but I do supply two pieces of customization: the Hack font, and the base16 color theme.

Hack Font

On OSX, fonts are now manageable via homebrew. To install the hack font, execute the following.

$ brew tap caskroom/fonts
$ brew cask install font-hack

Base16 Default Profile

In the OSX Terminal.app, “profiles” set many more configurable items beyond the 16-color pallette. These items include the shape of the cursor, opacity of the terminal background, audio and visual bell enablement, text encoding, and many more.

Critically, the profile may also declare the terminal as xterm-256color. The terminal configuration itself is, in fact, the correct place to make this declaration as opposed to the alternatives of declaring it either in .vimrc or .zshrc. Each of those applications are consumers of the terminal and have no business declaring the terminal’s color capabilities.

To make the default base16 color theme available to Terminal.app, I will need to download the appropriate .terminal file. To do this, simply execute the following.

git clone https://github.com/vbwx/base16-terminal-app.git

Now, go to Terminal/Preferences and find the Profiles tab. Choose “import” from the gear dropdown and find the “Default Dark.terminal” theme in the /profiles directory of the repo you just cloned.

Once this is done, you may delete the repository. To make this the default profile, simply choose the appropriate option under the “General” tab.

Dotfiles

With an eye toward repeatability and ease of configuration, I do as much as possible from the dotfiles, and keep the files themselves under version control. From there, configuring a net new development environment is as simple as cloning my dotfiles repository and symlinking .zshrc, .vimrc, and .tmux.conf from my home directory.

Specifically, the process involves two steps. First, Execute I execute something like the following from my home directory.

$ git clone https://github.com/jlbar/dotfiles.git

This creates the directory ~/dotfiles. Next, execute the following to define the appropriate symlinks.

$ ln -s ~/dotfiles/tmux/tmux.conf ~/.tmux.conf
$ ln -s ~/dotfiles/zsh/zshrc ~/.zshrc
$ ln -s ~/dotfiles/vim/vimrc ~/.vimrc

Now that the dotfiles are pulled and linked, we may discuss the details of configuring the look and feel of the CLI by visiting each component separately.

Zsh

I use oh-my-zsh plugins and themes to manage my zsh configuration. Relating specifically to look-and-feel, I use the excellent Bullet Train theme. Since Bullet Train is not one of the default oh-my-zsh themes, some additional work is required.

Specifically, oh-my-zsh allows for customization by defining the $ZSH_CUSTOM variable, which defaults to ~/.oh-my-zsh/custom. By default, git is set to ignore the custom directory, so that oh-my-zsh’s update process does not interfere with any user customizations.

To install the Bullet Train theme, create the /themes directory as follows.

$ mkdir $ZSH_CUSTOM/themes

Next, download the Bullet Train theme.

$ curl -O https://raw.githubusercontent.com/caiogondim/bullet-train-oh-my-zsh-theme/master/bullet-train.zsh-theme

Finally, configure the zsh theme in .zshrc in the typical fashion as follows.

ZSH_THEME="bullet-train"

These installation notes are condensed from the Bullet Train project’s README, located here:

https://github.com/caiogondim/bullet-train-oh-my-zsh-theme

Tmux

I use the tmux Maglev theme, which is designed to work with Bullet Train. Maglev requires the tmux plugin manager, tpm, located here:

https://github.com/tmux-plugins/tpm

Installation of tpm is straightforward and involves the creation of a ~/.tmux directory where plugins are stored.

Vim

For vim, as with zsh, I also use the base16 color theme, available here:

https://github.com/chriskempson/base16-vim

In keeping with the status line theme established in zsh with Bullet Train and in tmux with Maglev, in vim, I use the vim-airline status line, available here:

https://github.com/vim-airline/vim-airline
